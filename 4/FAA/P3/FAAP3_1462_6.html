AVISO: LOS RESULTADOS PUEDEN VARIAR SEGÚN LAS DISTINTAS EJECUCIONES, EL CÓDIGO TIENE UN FACTOR ALEATORIO DEBIDO A LA CREACIÓN ALEATORIA DE LAS PARTICIONES, LOS CRUCES Y MUTACIONES ALEATORIAS Y LA CREACIÓN DE UNA POBLACIÓN INICIAL ALEATORIA.¶

Detalles de implementación¶

Generación de la población inicial¶

La poblacin es un conjunto de individuos (tantos como se indique en el hiperparmetro tamPoblacion) que tienen cada uno un nmero de reglas arbitrario (desde 1 hasta un mximo que se pasa como hiperparmetro nReglas). As, la poblacin ser una lista de individuos y el individuo ser una lista de reglas, y cada regla es un cromosoma binario representado por una lista de bits aleatorios. La regla indica, para cada atributo, todos los posibles valores del mismo (esto lo sabemos gracias a nuestro diccionario de la P1, donde tenemos los n atributos y los ki valores distintos que puede tomar el atributo i-simo) que se encuentren activados en ese momento.

Cuando creamos la poblacin vamos a aprovechar tambin para crear un vector binario aleatorio (mismo tamao que el cromosoma) que es el que utilizaremos en el cruce implementado que se explica a continuacin.

Mecanismo de cruce¶

Hemos implementado un cruce uniforme, ya que como hemos visto en teora el cruce en un punto se utiliza ms bien cuando los distintos elementos que componen el cromosoma tienen relacin entre s y afectan al resultado final (por ejemplo, en el problema del viajante donde se representa el orden de visita de cada ciudad y este orden es importante). Por tanto, a la hora de cruzar dos individuos simplemente miramos el vector aleatorio para el cruce que tenamos generado y cada vez que aparezca un 1, cambiamos ese bit del individuo por el del otro individuo, y lo mismo con el otro, generando 2 descendientes que forman parte de la poblacin. Por ejemplo:

Individuo 1 --> 10011

Individuo 2 --> 00111

Vector cruce -> 10101

  • Descendiente 1 --> 00111
  • Descendiente 2 --> 10011

Destacar que este cruce se aplica a cada regla del individuo, pero como ambos pueden tener diferente nmero de reglas, se va a aplicar a las primeras r reglas de ambos individuos, siendo r el total de reglas del individuo que menos tenga.

Para elegir qu dos individuos cruzar, vamos recorriendo la poblacin y cruzamos proporcionalmente a pCruce, que es el hiperparmetro que determina la probabilidad de cruce.

Mecanismo de mutación¶

Hemos implementado la mutacin como un bit flip de cada bit de cada cromosoma de cada individuo de la poblacin. Este bit flip es proporcional a la probabilidad de mutacin que tenemos en el hiperparmetro pMutacion.

Mecanismo de clasificación¶

Para la clasificacin vamos a reaprovechar la funcin fitness que hemos usado en el entrenamiento. Durante el entrenamiento calculbamos el fitness de cada individuo de la poblacin para sacar el mejor individuo y realizar el proceso de seleccin proporcional a fitness, cruce, mutacin y elitismo durante cada poca, obteniendo el mejor individuo al final de todas. Para clasificacin, en cambio, le pasaremos a la funcin fitness el mejor individuo que ya obtuvimos en clasificacin, y adems un parmetro para que la funcin devuelva las predicciones (porque, por defecto, esta devuelve el fitness del individuo recibido). Tenemos dos tipos de clasificaciones que explicaremos ms adelante el motivo: una en la que si no coincide ninguna relga se predice una clase invlida (-1) y otra en la que se predice una clase aleatoria, y esto se realiza a travs del parmetro randomPred.

El mejor individuo para Titanic lo obtenemos con el clasificador de mximo 13 reglas, 50 individuos y 200 pocas y con el algoritmo mejorado prediciendo clases aleatorias cuando no se cumple ninguna regla.

Una regla se compone de 22 bits como los siguientes, y se muestra el significado de cada grupo: [XXX.YY.ZZZZZZZZZZZZZZZZ.C]

  • X: 1, 2 o 3 (PClass)
  • Y: male, female (Sex)
  • Z: 0, 1, 2, ... , 15 (Age). Se representan en orden lexicogrfico, es decir 0, 1, 10, 11, 12, 13, 14, 15, 2, 3...
  • C: 0 o 1 (Class)

As, por ejemplo, la regla [1, 0, 1, 0, 1, 1, 0, 0, 0, 0, 0, 1, 0, 1, 0, 1, 1, 1, 1, 1, 1, 0] significa que:

  • PClass debe valer 1 OR 3, AND
  • Sex debe valer male, AND
  • Age debe valer 0 OR 14 OR 2 OR 4 OR 5 OR 6 OR 7 OR 8 OR 9, AND
  • Class debe valer 0

El resto de reglas se interpretan igual. Es simplemente un One-Hot Enconder de los atributos donde cada bit a uno indica un valor activado del atributo, y si se activan varios del mismo atributo sera una OR.

import pandas as pd
from sklearn.metrics import confusion_matrix

from sklearn import metrics

df = pd.read_csv("titanic.data")
nombres_atrs = list(df.columns)
nombres_atrs.remove("Class")
X = df[nombres_atrs].values
y = df["Class"].values

# Utilizamos el propio sklearn para transformar los datos nominales a numricos
enc = OneHotEncoder(sparse=False)
enc.fit(X)
X2 = enc.transform(X)
x_train1, x_test1, y_train1, y_test1 = train_test_split(X2, y, test_size=0.3)

clf_laplace = MultinomialNB() # Con Laplace
clf_laplace.fit(x_train1, y_train1)

errores = 1 - cross_val_score(clf_laplace, X2, y, cv=10)
pred = cross_val_predict(clf_laplace, X2, y, cv=10)

matrizConfusionNB = confusion_matrix(y, pred)

La diagonal (Random Guess) divide el espacio ROC, los puntos por encima de la diagonal representan los buenos resultados de clasificacin (mejor que el azar), y los puntos por debajo de la lnea de los resultados pobres (peor que al azar).
